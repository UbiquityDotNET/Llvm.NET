Remarks

This is effectively an extension to ParserConfiguration adding control of the default options and directives. This is used with AppControlledDefaultsRootCommand to adapt the defaults otherwise forced by RootCommand. Of particular interest is the ResponseFileTokenReplacer which, if not set uses the default from ResponseFileTokenReplacer which is not publicly available. If the value is set to null then no replacer is used or supported. If it is set to a non-null value then that replacer is used.

The default values follows the default behaviors of the underlying library. This ensures the principle of least surprise while allowing for explicit overrides
