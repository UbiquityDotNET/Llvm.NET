Namespace Ubiquity.NET.Llvm.DebugInfo

This namespace contains all the support for the LLVM representation of debugging information.

Differences from previous release

A critical difference is that a Module does NOT own a DIBuilder. That idea in previous releases was a customized extension that was more accidental as a result of the former releases using object interning. However, once that was removed it was found that Module instances were attempting to hold fields or properties of things that were NOT part of the underlying native object. So, the pattern of use was changed to better match how the underlying LLVM API worked. In particular a DIBuilder is intended for short term use. It can (and does) own a DICompileUnit and it can reference the module it was created from. (Kaleidoscope Chapter 9 provides a sample of use in a visitor pattern where the instance is provided as a parameter to functions. This ability of providing a "context" was added to the visitor pattern specifically for this case.
